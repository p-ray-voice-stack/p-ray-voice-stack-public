# 开源与商业边界

这个仓库的公开部分服务于学习、验证和技术信任建立；商业协作服务于生产化交付、客户板卡适配和长期支持。

## 开源层包含

- `local-v2` 静态入门 demo
- 最小 FastAPI server base
- 官方 `ESP-VoCat v1.2` hardware-v2 baseline
- v3 public-preview contract 与 in-process runtime 示例
- 快速开始、v2/v3 对比和边界说明

这些内容应该足够帮助开发者理解项目方向，并复现最小公开路径。

## 商业层开始于

当团队需要以下能力时，就进入商业协作范围：

- 生产部署
- 私有化或托管运维
- 客户自有板卡适配
- 低延迟链路落地
- 定制音色、人格或交互体验
- 长期维护和迭代支持

## 为什么要划清边界

公开仓库应该真实、有用、可学习，但不需要把所有生产化经验一次性公开成完整交付包。

清楚的边界可以让两类用户都更舒服：

- 学习者知道自己能从公开仓库获得什么。
- To-B 团队知道什么时候需要更深入的交付支持。
